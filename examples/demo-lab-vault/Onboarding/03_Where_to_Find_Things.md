# 03 · Where to find things (the vault map)

How this vault is organized — the [framework](../../../framework/README.md) in one page.

## The layers
- **`Sources/`** — raw, immutable (papers, transcripts, data). Read, never edit.
- **`Wiki/`** — synthesized understanding (Concepts, Methodology). Agents write it; you read it.
- **`Output/`** — plans, drafts, filed-back explorations (private strategy).

## The daily surfaces (where things go)
- **`START_HERE.md`** — the daily inbox (🔴🟡🟢✅). What's new/pending.
- **`_handoff-log.md`** — claim a shared file (🔒), raise a handoff (🤝), log progress (📝).
- **`_Log.md`** — the append-only audit timeline.
- **`LANES.md`** — who's working what.

## Your space
- `Sub-tracks/02_flu/` — your track (START_HERE, Unknowns Register, day logs, questions.md).
- `progress-alex.md` — your worklog (feeds the check-in digest).

## The rule for "where does this go?"
Did an agent *write it as synthesis*? → `Wiki/`. Is it *raw material*? → `Sources/`. A today-update?
→ `START_HERE`. A handoff? → `_handoff-log`. (Full map: framework/documentation-and-handoffs.md.)
