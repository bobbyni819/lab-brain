The problem

Every lab has a mess, and a team — and tools solve for neither.

Years of data, code, decks, docs, grants and papers scatter across Box, Drive and laptops, named 1002732.pdf and 20260210_flu_ABM_BN.pptx. A new member takes months to learn what the lab already knows. And the moment two people — or two AI harnesses — touch the same shared notes, work gets clobbered. "Chat with my files" onboards a user. It cannot onboard a lab.

01 · the mess

Unreadable by design

Cryptic filenames, duplicated dumps, `_processed/` conventions, two domains interleaved, figures divorced from the code that made them. The knowledge is there — just not findable.

02 · the team

Contribution collides

Peers and undergrads each know a slice. Without a shared convention, concurrent edits overwrite each other and provenance evaporates. Onboarding a whole team is a coordination problem, not a chat.

The approach

Three moves that turn storage + people into one brain.

move 01

Onboard fast

One command wires a newcomer's Claude Code and Claude Science to the shared KB, with the lab's conventions preloaded as skills. Zero to connected in minutes.

move 02

Contribute, don't just read

A swarm of area-specialist agents reads each artifact in full, verifies, and writes to the KB with provenance and a confidence — never a skim.

move 03

A whole team, one brain

Peers and mentees contribute concurrently through a shared framework of conventions — where things go, how figures are made, how mentees report up. Additive, no clobbering. That framework is the hero.

The core · your mess → your brain

Point it at a lab's files. It reads everything, and files each where it belongs.

Code, slides, docs, grants, data, papers — however cryptic or disorganized. A cheap structural sort (runnable today, no LLM) buckets every file, catches duplicates, and drops non-science; then a swarm of area-specialist agents reads each file in full and routes what it learns into the shared KB. The framework · the hero

"Where does this go?" — answered by default.

A lab loses knowledge not because people don't write things down, but because nobody agrees on where things go — so updates, decisions, and findings scatter into chat logs and vanish. The framework gives every artifact one home. Pick something you just did.

Six conventions (+ two)

Generalized from a system a real lab already runs.

01 · files

KB structure

Three layers — Sources (raw, immutable) · Wiki (agent-owned synthesis) · Output (private strategy) — and one filing rule so evidence and interpretation never blur.

02 · names

Naming & notes

A trailing-YYYY-MM-DD convention so filename-sort = sense, plus one YAML note template every file starts from.

03 · docs

Documentation & handoffs

The routing map above: START_HERE daily-brief (🔴🟡🟢✅), _Log, the _handoff-log board, MEMORY.

04 · figures

Figures & findings

Restyle, never reinvent; verify by looking. Scripts read real data (no hard-coded numbers), and FIGURE_FINDINGS.md logs what each panel shows. Captions are claims, not labels.

05 · storyline

Storyline & manuscript

Messages first, figures follow. A per-figure storyline doc where every panel earns a “📖 Beat” in the argument, kept in sync with the prose.

06 · onboarding

Onboarding a member

A walkable sequence + a per-person scaffold with supervision gates and an Unknowns Register that extracts the mentor's tacit knowledge instead of losing it.

+ mentorship — the mentee↔mentor check-in loop (next section)  ·  + harness playbook — Claude Science & Claude Code on one KB: one harness owns a working-dir at a time.

Depth · scales to the work

The number of agents adapts to the corpus.

Nobody hand-tunes it. A 200-file lab gets a handful of readers; a 12,000-file lab gets tens, fanned out concurrently — one specialist per content area, bounded so it never runs away. Drag the corpus size.

Multi-user · already in production

Peers work as equals. Mentees get scaffolded — and report up.

The roster assigns each member a role, and the collaboration shape follows. The real engine is a paired-by-date check-in loop: a neutral -update (what shipped) kept separate from a structured -feedback (the mentor's voice). Flip the mode.

One showcased capability

It even reads the figures — and stays honest.

Already in the lab

"We don't just propose this — we're onboarding our own team onto it."

The collaboration model is generalized from the coordination practices the Hickey Lab runs day-to-day on a multi-person spatial-omics project — CODEX spatial proteomics × MALDI-IMS lipidomics on human testis — plus a Data+ undergraduate team. The practice is real; Lab Brain packages it so any lab can adopt it — and we're adopting this version ourselves.
